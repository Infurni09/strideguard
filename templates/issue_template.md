# StrideGuard Issue Template — Agent Instructions

This file documents the format StrideGuard uses when calling `gitlab:create_issue`.
The agent reads this as a reference — it is not processed as a Liquid template.
All variable substitution is performed by the agent inline when constructing the API call.

---

## Issue title format

```
[STRIDE-<CATEGORY_INITIAL>-<N>] [<SEVERITY_UPPERCASE>] <imperative title in 10 words or fewer>
```

Examples:
- `[STRIDE-T-1] [HIGH] SQL injection via unsanitized order_id parameter`
- `[STRIDE-I-2] [HIGH] Hardcoded Stripe secret key in source code`
- `[STRIDE-S-1] [MEDIUM] No authentication on /api/payments/initiate endpoint`
- `[STRIDE-D-1] [LOW] Missing rate limiting on payment initiation endpoint`

---

## Issue body format

The agent constructs the issue body using the following structure, substituting
all placeholders with real values from the threat record:

```markdown
## Threat summary

**ID:** `STRIDE-<CATEGORY_INITIAL>-<N>`
**Category:** <Spoofing | Tampering | Repudiation | Information Disclosure | Denial of Service | Elevation of Privilege>
**Severity:** <critical | high | medium | low>
**Component:** `<file path or service name>`
**CWE:** <CWE-NNN or N/A>
**Detected in MR:** !<mr_iid> — <mr_title>

---

## Description

<2–4 sentences describing: what the threat is, exactly where in the code it lives
(file name, function name, line number if known), and how an attacker could exploit it>

---

## Remediation

<2–4 sentences of specific, actionable steps to fix the vulnerability. Include
code patterns, library names, or configuration changes where applicable>

---

## References

- [CWE-NNN: <CWE title>](https://cwe.mitre.org/data/definitions/NNN.html)
  *(omit this line if no CWE applies)*
- [STRIDE Threat Modeling — Microsoft](https://learn.microsoft.com/en-us/azure/security/develop/threat-modeling-tool-threats)
- [OWASP Threat Modeling Cheat Sheet](https://cheatsheetseries.owasp.org/cheatsheets/Threat_Modeling_Cheat_Sheet.html)

---

## Resolution checklist

- [ ] Understand the threat and confirm it applies to this codebase
- [ ] Implement the recommended remediation
- [ ] Add or update tests covering the security scenario
- [ ] Request re-review from the security team (if critical or high severity)
- [ ] Close this issue with a reference to the fixing commit

---

*Generated by [StrideGuard](https://github.com/Infurni09/StrideGuard) — AI Threat Modeling Agent*
*Triggered by: <trigger type> at <ISO 8601 timestamp>*
```

---

## Labels to apply

The agent applies these labels on every issue it creates:

| Label | Purpose |
|-------|---------|
| `strideguard` | Identifies all StrideGuard-generated issues |
| `security` | Integrates with existing security workflows |
| `stride::spoofing` | STRIDE category (use the matching label) |
| `stride::tampering` | — |
| `stride::repudiation` | — |
| `stride::information-disclosure` | — |
| `stride::denial-of-service` | — |
| `stride::elevation-of-privilege` | — |
| `severity::critical` | Severity (use exactly one) |
| `severity::high` | — |
| `severity::medium` | — |
| `severity::low` | — |

Only one `stride::*` label and one `severity::*` label are applied per issue.

---

## Closing a resolved issue

When a threat is no longer present in an updated MR diff, the agent calls
`gitlab:close_issue` and adds this closing note:

```
Resolved in MR !<mr_iid> — this threat is no longer present in the updated diff.
Closed automatically by StrideGuard.
```
